Селекторы. Специфичность селекторов

И так, существует 5 основных типов селекторов.
Каждый селектор имеет свой приоритет (вес). И именно вес определяет, стили какого селектора будут применяться.

!important Устанавливает наиболее высокий приоритет стиля. Обращение происходит через атрибут !important
inline Задает стиль елементу, не обращаясь к CSS. Обращение происходит через атрибут style="..." в самом теге
id Задает стиль елементу, путем присваивания селектора id"...". Повторение id недопустимо.
class Задает стиль елементу, путем присваивания селектора class="...". Повториние class возможно в неограниченном колличестве.
tag Селектор по названию Тега.

Рассмотрим поведение селектора по названию тега.

Этот абзац был создан с помощью тега (р).
Тегу (р) через css применено свойство (color:) со значеним (red).
Выглядит это вот так:

css
p { color: red }

И соответственно текст стал красным.
Следует отметить, что селекторы по названию тега это селекторы с самым низким приоритетом.

Рассмотрим поведение селектора по названию класа.

Для написания этого абзаца сново используем тег (р) с ранее примененным стилем ( p { color: red; })

Присвоив тегу (р) селектор (class="blue_text"), мы изменили цвет текста на синий.
Выглядит это вот так:

html
< p class="blue_text">....< /p>

css>
p { color: red }
.blue_text { color: blue; }

Текст стал синим, потому что приоритет (вес) селектора по названию класа (class) выше приоритета селектора по названию тега (tag).

Рассмотрим поведение селекторов по id.

Сново тег (р) со всеми ранее примененными стилями.

Присваиваем тегу (р) еще один селектор (id="green_text").

html
< p class="color_blue" id=green_text">....< /p>

css
p { color: red }
.blue_text { color: blue; }
#green_text { color: green; }

И цвет текста меняется на зеленый.
Принцип тот же. Применяется свойство того селектора, чей приоритет (вес) больше.
Приоритет селектора по id > class > tag

Рассмотрим поведение селекторов inline.

Сново тег (р).
Ранее примененные стили используются. И как видим цвет текста сново изменился.
Это было сделано путем присвоения селектора inline.
Этот селектор присваивается тегу в качестве атрибуа style="", без обращения к css.

html
< p class="blue_text" id="green_text" style=" color:rgb(118, 170, 170); " >

И соответственно цвет стал мятным, потому что приоритет (вес) селектора inline выше чем у остальных.

Рассмотрим поведение селектора !important.

Сново тег (р).
Все ранее примененные стили так же используются.

Отличительная особенность селектора important в том, что он не задает новый стиль с более высоким приеритетом,
он устанавливает самый высокий приоритет для стилей любого другого селектора, при этом совершенно не важно какой у него (вес).

Посмотрим как это работает.

Создаем новый селектор класа .text_dark_red со свойством и занчением { color: red !important; } и присваеваем его тегу (р)

css
p { color: red }
.blue_text { color: blue; }
#green_text { color: green; }
.text_dark_red color: red !important;

html
< p class="text_dark_red blue_text" id="green_text" style="color:rgb(118, 170, 170); ">

Как видим текст стал темно-красным.
Не смотря на то что у тега (р) есть селекторы по id и inline селекторы, к нему всеровно применились стили селектора класа.

Рассмотрим пример использования нескольких класов.

К любому елементу можно применить несколько класов, пречисляя их в атрибуте class через пробел.
Подобное применение используется в примере выше.
< p class="text_dark_red blue_text" id="green_text" style="color:rgb(118, 170, 170); ">
По скольку оба класса имеют одинаковые стилевые свойства, применятся значения свойств того класса, который стоит последний в очереди, или того у кого выше приоритет.
В нашем случаи применились свойства класса text_dark_red.
Все дело в том что у класса text_dark_red выше приоритет, так как к нему применен селектор !important.
Посмотрим подробнее как это работает.

Создаем несколько новых классов и пропишем для них стили.

css
.class_bold { font-weight: 700; font-size: 22px; }
.class_italik { font-style: italic; font-size: 18px; }
.class_lite {} font-weight: 100; color: green; }

Создаем (р), применяем к нему class_bold и style="color: black" (а то красный както вооще не читается), и пишем произвольный текст.
Выглядит это вот так.

html
< p class="class_bold" style="color: black"> .... < /p>

Вот что получилось.

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Illo ut cumque iste omnis non necessitatibus, voluptas recusandae ab sit qui ad accusamus porro. Nesciunt exercitationem aliquam cupiditate debitis inventore beatae deserunt, impedit excepturi atque, minima maiores nulla, placeat rerum enim.

Как видим текст стал черным, жирным и изменился его размер (22 рх). Стили применились.

Подключаем еще один класс.

html
< p class="class_bold class_italik" style="color: black"> .... < /p>

Вот что получилось.

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Illo ut cumque iste omnis non necessitatibus, voluptas recusandae ab sit qui ad accusamus porro. Nesciunt exercitationem aliquam cupiditate debitis inventore beatae deserunt, impedit excepturi atque, minima maiores nulla, placeat rerum enim.

Как видим изменилось начертание текста (он стал курсивным), и изменился его размер (18 рх), можно считать что стили класса class_italik полностью применились.
Но свойство font-weight: 700 (жирный текст) который был задан классом class_bold, по прежнему работает.
Все потому что у класса class_italik нету свойства font-weight которое бы заменило это свойство у класса class_bold.
Вообщем, думаю принцып понятен.

Подключаем еще один класс.
html
< p class="class_bold class_italik class_lite" style="color: black"> .... < /p>

Вот что получилось.

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Illo ut cumque iste omnis non necessitatibus, voluptas recusandae ab sit qui ad accusamus porro. Nesciunt exercitationem aliquam cupiditate debitis inventore beatae deserunt, impedit excepturi atque, minima maiores nulla, placeat rerum enim.

Как видим некоторые свойства класса class_lite применились а некоторые нет.
Применилось свойство (font-weight: 100), он стал тонким.
Но свойство (color: green;) все же не применилось.
Все потому что черный цвет текста задан inline селектором, а приоритет inline селекторов выше чем у селекторов по классам.

А теперь рассмотрим пример вложенных селекторов.

Создаем (р) параграф с текстом. Создаем в нем несколько ссылок. И еще одну ссылку вне параграфа.
Тегу (р) присваиваем класс text_random, а всем ссылкам класс link.
Классу link говорим что он должен быть font-size: 22px.
Выглядит это вот так.

html
< p class="text_random">....text....< a href="#" class="link">..Link 1..< /a>...text...< a href="#" class="link">...Link 2...< /a>...< /p>
< a href="#" class="link">..Link 3...< /a>

css - шаг 1
.link { font-size: 22px; }

css - шаг 2
.text_random .link { font-size: 18px; }

И вот что получилось.

Lorem ipsum Provident ratione et dolor sit amet consectetur adipisicing elit. Illo facilis id molestiae sit porro nihil blanditiis obcaecati, adipisci exercitationem recusandae! Provident ratione et consequuntur laboriosam quasi qui possimus laborum neque laudantium? Corporis voluptates repellat minima aspernatur cumque error accusamus dolorum hic deserunt dolorem impedit ullam soluta, molestias atque temporibus fuga!

ossimus labor

Как видим размер ссылок в параграфе - 18рх, а размер ссылки после параграфа - 22рх.
Хотя всем трем ссылкам присвоен одинаковый класс - class="link".

Вот как следует понимать селектор .text_random .link { font-size: 18px; }

Ко всем елементам с калссом .link которые расположенны в елементах с калссом .text_random применяем font-size: 18px;
Вот и все.
Именно так работают селекторы по вложенности.
